# 보안 정책 (Security Policy)

## 지원 버전

최신 릴리스에 대해서만 보안 수정을 제공합니다.

| 버전 | 지원 |
| ---- | ---- |
| 0.1.x | ✅ |
| < 0.1 | ❌ |

## 취약점 제보

보안 취약점은 **공개 이슈로 올리지 말아 주세요.** 비공개로 제보해 주시면 신속히 대응하겠습니다.

- 이메일: **zihado@gmail.com**
- 또는 GitHub의 [Private vulnerability reporting](https://github.com/rescenedev/alldoc/security/advisories/new) 사용

제보 시 다음을 포함해 주세요.

- 취약점 유형과 영향 범위
- 재현 단계(PoC 가능하면 첨부)
- 영향을 받는 버전 / 환경(macOS 버전 등)

## 대응 절차

1. **확인**: 영업일 기준 3일 이내 접수 확인 회신.
2. **분석**: 심각도 평가 및 영향 범위 파악.
3. **수정**: 비공개로 패치 개발 → 검증.
4. **배포**: 수정 릴리스 공개, 제보자 크레딧(원하실 경우).

가능한 한 빠르게(심각도에 따라) 패치를 배포하는 것을 목표로 합니다.

## 앱의 보안 특성 (참고)

- AllDoc 은 **로컬 전용** 앱입니다. 문서 내용·인덱스·즐겨찾기는 모두 사용자 기기에만 저장되며 외부로 전송하지 않습니다.
- 인덱스(SQLite FTS5)와 설정은 앱 지원 디렉터리 / `UserDefaults` 에 보관됩니다.
- 파일 접근은 사용자가 **명시적으로 추가한 폴더**로 한정됩니다.
- 외부 도구(`fd`, `fzf`, `pdftotext`)를 서브프로세스로 호출하며, 인자는 파일 경로/검색어로 한정됩니다.

## 배포물 무결성

- 정식 릴리스는 [GitHub Releases](https://github.com/rescenedev/alldoc/releases) 에 게시되며, Homebrew 포뮬러가 `sha256` 체크섬을 검증합니다.
- 현재 바이너리는 ad-hoc 서명 상태입니다(개발자 인증서 미적용). 신뢰할 수 있는 출처(공식 릴리스/Homebrew tap)에서만 설치하세요.
